# 混合检索与精排

RAG 为什么需要 Rerank？因为第一阶段召回强调“尽量不要漏掉相关内容”，返回的候选 Chunk 中通常会混入噪声。Rerank 会再次比较用户问题与候选片段的相关性，把证据更充分的内容排到前面，从而提高最终上下文质量。

BM25 擅长关键词精确匹配，例如模型名、函数名和专业术语；向量检索擅长语义相似匹配，可以找回措辞不同但含义接近的内容。混合检索同时使用两者，减少只依赖单一路径造成的漏召回。

# RRF 融合

BM25 分数与向量距离不在同一尺度，不能直接相加。Reciprocal Rank Fusion（RRF）只使用每一路的排名，通过 `1 / (k + rank)` 累加融合分数，因此实现简单、稳定，也方便解释。

# 引用核查

ContextBuilder 为本轮检索到的 Chunk 分配 C1、C2 等编号。CitationChecker 只接受本轮 citation_map 中存在的引用，模型输出的 C99 等未知编号会被过滤。
